        <p class="eyebrow">Operating path</p>
        <h2>What changes when CALYPR becomes the operating layer for research delivery.</h2>
        <p>
          Instead of building separate processes for storage, metadata, compute, and downstream packaging, teams get one operating path that carries data and context forward together.
        </p>
      </div>
      <div class="solution-roadmap">
        <article class="solution-roadmap__step">
          <span>01</span>
          <strong>Standardize access</strong>
          <p>Replace ad hoc file movement with DRS-native registration, scoped remotes, presigned transfer, and collaborator-aware policy boundaries.</p>
        </article>
        <article class="solution-roadmap__step">
          <span>02</span>
          <strong>Structure the context</strong>
          <p>Map tabular project information into FHIR-shaped resources and graph relationships that can be validated, reused, and queried.</p>
        </article>
        <article class="solution-roadmap__step">
          <span>03</span>
          <strong>Operationalize execution</strong>
          <p>Run repeatable workflows through a portable execution layer instead of rebuilding task operations around each environment.</p>
        </article>
        <article class="solution-roadmap__step">
          <span>04</span>
          <strong>Carry assets forward</strong>
          <p>Keep data, workflow provenance, benchmark context, and model artifacts inside one governed operating system.</p>
        </article>

  <section class="solution-section">
    <p class="eyebrow">What sits underneath</p>
    <h2>The story is backed by technical capabilities your platform team can verify.</h2>
    <div class="value-grid">
      <article>
        <h3>Governed object access</h3>
        <p>Syfon exposes GA4GH DRS APIs, presigned upload and download URLs, and multipart transfer lifecycle support so data operations have a real service contract behind them.</p>
      </article>
      <article>
        <h3>Version-aware data workflows</h3>
        <p>Git-DRS stores pointer files in Git, keeps large binaries out of normal history, and connects repositories to scoped remotes for push, pull, hydration, and delete workflows.</p>
      </article>
      <article>
        <h3>Structured metadata and discovery</h3>
        <p>Forge validates and publishes metadata, CALYPR uses FHIR resource structure for research entities, and GRIP provides graph-oriented query patterns across integrated project data.</p>
      </article>
    </div>
